## 简介

公共项目不是个人笔记，每一次贡献都值得被认真对待。

本项目提供一套**通用的、可实践的**开源项目贡献流程标准，适用于所有开源项目，不局限于特定社区或技术栈。无论你是第一次参与开源的新手，还是希望规范团队协作流程的项目维护者，都可以从中获益。

## 内容涵盖

| 章节 | 说明 |
|------|------|
| 为什么需要贡献规范 | 公共项目与个人项目的区别 |
| 认识开源项目结构 | 仓库首页、关键文件、GitHub 功能区域 |
| Fork 工作流 | Fork → Clone → Branch → Commit → Push → PR 完整流程 |
| Commit 规范 | Conventional Commits、AI 辅助撰写 |
| Pull Request 规范 | PR 标题、描述、Code Review 流程 |
| 项目配置最佳实践 | Issue 模板、PR 模板、CI/CD、Dependabot、Release Notes |
| 所有权转移 | GitHub Transfer ownership |
| Fork 站点预览 | Fork 后的文档站点部署 |
| SEO 与搜索引擎 | 文档框架的 SEO 配置 |
| 常见错误与反面模式 | 直接推 main、不写描述、不同步上游等 |

## 本地开发

```bash
# 安装依赖
npm install

# 启动开发服务器
npm run docs:dev

# 构建
npm run docs:build
```
